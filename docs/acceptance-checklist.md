# 验收标准 (Acceptance Checklist)

本项目在交付前需严格符合以下验收标准。

## 1. 基础验收
- [ ] `npm install` 成功执行，无严重的版本依赖错误。
- [ ] `npm run dev` 成功启动，能顺利访问本地开发服务器。
- [ ] `npm run build` 成功，无任何 TypeScript 类型检查报错。
- [ ] 浏览器控制台 `Console` 干净，无阻塞性错误（如 Uncaught Type Error, React key warnings 等）。

## 2. 玩法验收
- [ ] **掉落物生成**：能随时间间隔稳定生成，旋转及掉落速度符合当前关卡（DifficultyConfig）配置。
- [ ] **交互切割**：手势食指或鼠标调试拖动均能准确判定切中物件。
- [ ] **信息反馈**：切中后，屏幕中央和对应的位置能够正确显示所切中的“学科名”和“物件名”。
- [ ] **喜爱/厌恶规则**：喜爱学科能带来高额加分及减免厌恶次数；厌恶学科会带来扣分和厌恶次数增加，连续切中存在额外惩罚。
- [ ] **结束逻辑**：单局游戏厌恶次数达到 5 次，或切中 2 次致命炸弹，游戏能准确跳转结束面板并出示最终成绩。

## 3. 体验验收
- [ ] **摄像头异常降级**：当用户拒绝授权摄像头或在非 HTTPS 环境下时，系统不会白屏，并给出友好的错误提示，提供一键进入“鼠标调试模式”的按钮。
- [ ] **图片资源降级**：当对应的 PNG 和 SVG 资源未部署或加载失败时，Canvas 内的对象自动显示 Emoji，不会引起游戏崩溃或阻塞。
- [ ] **音频异常降级**：音频文件加载失败时，游戏依然保持运行，不会因为静音错误导致交互中断。
- [ ] **UI 易用性**：JARVIS 风格的 HUD 面板信息清晰，能看清当前模式、FPS、关卡倒计时以及分数。
- [ ] **规则可读性**：规则页包含完整的规则说明，底部“学科物件数据库”展示清晰，能在一屏或滚动后看清所有 12 门学科对应的物件和 Emoji。
