# 学科忍者：知识乱斗 测试计划 (Test Plan)

本测试计划用于在每次发版前进行手动核心功能验证，确保游戏流程、手势识别及核心计分逻辑无严重回归问题。

## 一、页面流程测试

1. [ ] 首页加载正常，点击“启动全息手势模式”能进入规则页。
2. [ ] 规则页点击“我已了解，开始手势校准”能进入校准页。
3. [ ] 校准页左右手测试通过后（或点击鼠标调试模式），能进入关卡介绍页。
4. [ ] 关卡介绍页倒计时结束后（设定为数秒内），能自动进入游戏页。
5. [ ] 在游戏页中，厌恶次数累计达到 5 次或切中 2 次炸弹后，能正确跳转到游戏结束页。
6. [ ] 游戏结束页点击“重新开始”，能返回首页，并且历史数据（分数、命中数、倒计时等）彻底清空。

## 二、摄像头测试

1. [ ] **允许权限**：初次进入校准页时，浏览器弹出授权提示，点击允许后，背景正常显示摄像头画面。
2. [ ] **拒绝权限**：浏览器拒绝权限后，页面不白屏，并显示红色错误提示“无法访问摄像头”，允许直接点击进入鼠标调试模式。
3. [ ] **环境检测**：在非 `localhost` 或非 `HTTPS` 下运行，页面提示摄像头报错，功能优雅降级。
4. [ ] **模型加载异常**：若断网导致 MediaPipe 加载失败，页面提示“引擎加载失败”，不会白屏。
5. [ ] **无摄像头体验**：使用鼠标调试模式能完整游玩整个流程。

## 三、手势测试

1. [ ] **左手识别**：举起左手，画面正确出现青色骨骼线，食指指尖带高亮，手腕处显示“左手”。左手划过掉落物能成功切中。
2. [ ] **右手识别**：举起右手，画面正确出现青绿色骨骼线，食指指尖带高亮，手腕处显示“右手”。右手划过掉落物能成功切中。
3. [ ] **双手并行**：双手同时出现在画面中，两条轨迹各自独立计算碰撞，不发生串键。
4. [ ] **镜像对齐**：视频画面为自拍镜像，手势坐标系与视频画面完全对齐贴合，无偏移现象。

## 四、计分测试
*(可通过控制台执行 `window.runScoreSystemSelfTest()` 快速验证核心算法)*

1. [ ] **普通学科**：切中获得 +1 分。
2. [ ] **喜爱学科（无厌恶）**：当前厌恶次数为 0 时，切中获得 +3 分。
3. [ ] **喜爱学科（有厌恶）**：当前厌恶次数 > 0 时，切中获得 +2 分，并抵消（减去）1 次厌恶次数。
4. [ ] **厌恶学科（单次）**：切中获得 -3 分，厌恶次数 +1。
5. [ ] **厌恶学科（连续）**：连续切中同一门厌恶学科获得 -8 分，厌恶次数 +3，触发强红色闪屏警告。
6. [ ] **火焰（第4关起）**：切中获得 -5 分，视为危险命中，增加 1 次厌恶次数。
7. [ ] **炸弹（第5关起）**：切中 2 次炸弹立即触发游戏结束逻辑。

## 五、性能测试

1. [ ] **帧率监控**：HUD 上的 FPS 在普通电脑浏览器中应稳定在 40-60 左右，无长期掉帧变红现象。
2. [ ] **数量上限**：HUD 显示的“掉落物”数量不超过当前关卡上限（如第一关 28），“粒子数”不超过设定的 300 限制。
3. [ ] **内存泄漏**：在主游戏页与结束页之间来回切换 5 次，内存占用和粒子数组长度不会无限增长。
4. [ ] **事件清理**：页面切换或离开时，旧的 `requestAnimationFrame` 被正确停止，无控制台循环报错。
