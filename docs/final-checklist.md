# 最终交付检查清单

在项目上线或最终交付前，请逐一核对以下清单以确保游戏质量与稳定性。

## 基础运行
- [ ] `npm install` 成功，无严重依赖冲突报错。
- [ ] `npm run dev` 成功启动，能顺利访问本地开发服务器。
- [ ] `npm run build` 成功，TypeScript 编译通过，无类型报错。
- [ ] `npm run preview` 成功，打包后的产物能正常在本地预览。

## 页面流程
- [ ] 开始页（StartScreen）正常渲染，HUD 风格 UI 无变形。
- [ ] 规则页（RulePanel）正常展示，能完整阅读规则和学科物件图鉴。
- [ ] 校准页（StartCalibration）正常请求摄像头并加载手势模型。
- [ ] 关卡介绍页（LevelIntroPanel）正常显示，能正确展示随机抽取的喜爱/厌恶学科及当前关卡难度参数。
- [ ] 游戏页（GameCanvas 等）正常运转，HUD、掉落物、手势、背景视频层级正确。
- [ ] 游戏结束页（GameOverPanel）正常弹出，能正确展示最终得分和关卡数。

## 核心功能
- [ ] 摄像头能成功打开并铺满背景。
- [ ] 双手能被 MediaPipe 准确识别。
- [ ] 手部骨骼能实时绘制并贴合实际画面，镜像坐标转换正确。
- [ ] 掉落物能根据设定的时间间隔和速度生成、下落、旋转并自动销毁。
- [ ] 玩家挥动双手食指能准确切中掉落物。
- [ ] 在无摄像头或按下 `D` 键时，鼠标调试模式能切中掉落物。
- [ ] 每次切中后，屏幕中央能弹出正确的反馈（学科名、物件名、分值）。
- [ ] 分数变化符合复杂规则（喜爱加分/抵消、厌恶扣分/增加厌恶次数、连续厌恶重罚）。
- [ ] 厌恶次数达到 5 次或触发了炸弹，游戏会立即判定结束。
- [ ] 点击“重新开始”后，分数、命中数、时间、场上残留掉落物、粒子效果均能彻底清空。

## 资源降级
- [ ] 真实的 PNG 图片不存在时，能自动降级为 SVG，SVG 也不存在时能优雅降级显示 Emoji，且不导致页面崩溃。
- [ ] 音频文件不存在或加载失败时，游戏静音运行，不阻塞渲染主循环。
- [ ] MediaPipe 模型加载失败或网络波动时，页面不白屏，能抛出友好的错误提示。
- [ ] 摄像头被拒绝时，能显示报错并允许玩家无缝使用鼠标调试模式体验游戏。

## 性能与稳定性
- [ ] FPS 监视器数值正常（推荐在普通设备上保持 40-60 之间）。
- [ ] 各个页面切换之间无明显卡顿，内存无明显飙升。
- [ ] 重新开始游戏后，没有残留的 `requestAnimationFrame` 导致逻辑被重复执行。
- [ ] 浏览器控制台 `Console` 干净，无 React 的 Key 报错、空指针异常或其他阻塞性错误。
- [ ] `useEffect` 中的定时器、事件监听器均在组件卸载时得到了正确的清理。
