# 学科忍者：知识乱斗

## 项目简介
这是一个基于 React + TypeScript + Vite + MediaPipe 的前端互动游戏。玩家通过摄像头前的双手食指手势切中屏幕中随机掉落的学科物件，系统会根据普通学科、喜爱学科、厌恶学科进行计分。

## 技术栈
1. React
2. TypeScript
3. Vite
4. MediaPipe Tasks Vision
5. HTML Canvas
6. Three.js / React Three Fiber，作为后续视觉增强预留
7. 原生 CSS

## 核心功能
1. 摄像头背景。
2. MediaPipe 双手识别。
3. 手部骨骼绘制。
4. 左右手食指切割掉落物。
5. 鼠标调试模式。
6. 学科物件随机掉落。
7. 喜爱 / 厌恶学科规则。
8. 连续厌恶惩罚。
9. 关卡系统。
10. 音效和背景音乐。
11. JARVIS 风格 HUD。
12. 游戏结束结算。

## 常见问题排查 (摄像头与识别)

1. **摄像头打不开：**
   * 检查浏览器是否允许摄像头权限。
   * 检查是否使用 HTTPS 协议或 localhost 本地环境（浏览器的安全策略限制）。
   * 检查摄像头是否被其他软件占用。

2. **手势识别不准确：**
   * 保持环境光线充足。
   * 确保双手完整入镜。
   * 避免背景过于杂乱。
   * 建议玩家距离摄像头 0.8–1.5 米左右。

3. **游戏卡顿：**
   * 关闭其他占用摄像头的软件或沉重的后台程序。
   * 降低同屏物件数量限制。
   * 减少爆炸粒子数量。
   * 建议使用 Chrome / Edge 最新版浏览器获得最佳性能。

4. **没有摄像头：**
   * 可以按下键盘 `D` 键（或在无摄像头时默认开启）使用鼠标调试模式继续体验。

## 在线游玩说明

如果你收到的是打包部署后的公网 HTTPS 链接，请参考以下流程游玩：

1. 打开部署后的 HTTPS 链接。
2. 建议使用电脑端 Chrome / Edge 浏览器获得最佳性能。
3. 点击首页的“启动全息手势模式”。
4. 仔细阅读规则页（支持滚动）。
5. 浏览器提示时，**允许摄像头权限**。
6. 举起双手完成手势实战校准。
7. 用食指切中掉落物件，体验体感互动的乐趣！
8. **兜底方案**：如果摄像头不可用（或处于非 HTTPS 环境报错），可以直接点击首页或校准页上的“进入鼠标调试模式”来用鼠标拖动游玩。

## 演示建议 (Presentation Guide)
为了获得演示效果，建议参考以下操作：
1. **浏览器推荐**：优先使用 Chrome 或 Edge 最新版本，以获取最优的 Canvas 和 MediaPipe 性能。
2. **演示模式**：在首页点击“进入演示模式”按钮，系统会自动模拟手势切中掉落物。该模式无需摄像头，非常适合**作品集录屏**或向观众快速展示核心计分规则和粒子爆炸效果。
3. **环境兜底**：如果网络或光线环境极差导致手势识别失败，请灵活使用“鼠标调试模式”来保证展示的流畅度。
4. **展示流程推荐**：首页项目信息 → 规则页快速浏览 → 演示模式试玩 1 关 → 退回首页 → 启动全息手势模式（展示真实摄像头互动） → 故意触发红屏炸弹展示游戏结束面板。
